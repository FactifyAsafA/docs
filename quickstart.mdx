# Quickstart Guide

Get up and running with the Factify API in minutes. This guide will walk you through creating your first document, managing collaborators, and generating an entry page.

## Prerequisites

Before you begin, make sure you have:

- A Factify account ([Sign up here](https://factify.com/signup))
- Your API key from the [API Keys section](https://factify.com/settings/api-keys)
- A PDF file to upload 

## Step 1: Set Up Authentication

First, let's set up your API key. You can use it in several ways:

### Using cURL

```bash
export FACTIFY_API_KEY="your_api_key_here"
```

### Using JavaScript/Node.js

```javascript
const apiKey = 'your_api_key_here';
const baseURL = 'https://api.factify.com';

const headers = {
  'Authorization': `Bearer ${apiKey}`,
  'Content-Type': 'application/json'
};
```

### Using Python

```python
import requests

api_key = 'your_api_key_here'
base_url = 'https://api.factify.com'
headers = {
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json'
}
```

<Note>
Keep your API key secure and never commit it to version control. Use environment variables in production.
</Note>

## Step 2: Create Your First Document

Let's upload a document to Factify. You can upload various file types including PDFs, Word documents, text files, and images.

### Using cURL

```bash
curl -X POST https://api.factify.com/documents \
  -H "Authorization: Bearer $FACTIFY_API_KEY" \
  -F "payload=@/path/to/your/document.pdf" \
  -F "name=My First Document" \
  -F "description=A sample document for testing the Factify API"
```

### Using JavaScript/Node.js

```javascript
const formData = new FormData();
formData.append('payload', fileInput.files[0]); // Your file
formData.append('name', 'My First Document');
formData.append('description', 'A sample document for testing the Factify API');

const response = await fetch(`${baseURL}/documents`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`
  },
  body: formData
});

const document = await response.json();
console.log('Document created:', document);
```

### Using Python

```python
import requests

# Prepare the file and metadata
files = {
    'payload': ('document.pdf', open('/path/to/your/document.pdf', 'rb'), 'application/pdf')
}
data = {
    'name': 'My First Document',
    'description': 'A sample document for testing the Factify API'
}

response = requests.post(
    f'{base_url}/documents',
    headers={'Authorization': f'Bearer {api_key}'},
    files=files,
    data=data
)

document = response.json()
print('Document created:', document)
```

### Response

You'll receive a response like this:

```json
{
  "id": "doc_123456789",
  "name": "My First Document",
  "description": "A sample document for testing the Factify API",
  "current_version_id": "ver_987654321",
  "created_at": "2024-01-15T10:30:00Z",
  "created_by": {
    "object": "user",
    "id": "user_123",
    "name": "John Doe"
  },
  "status": "done",
  "url": "https://factify.com/documents/doc_123456789"
}
```

<Note>
The document status will be "processing" initially and change to "done" once the upload and processing is complete.
</Note>

## Step 3: Invite Collaborators

Now let's invite some collaborators to work on your document. You can invite users by email or user ID.

### Invite by Email

```bash
curl -X POST https://api.factify.com/documents/doc_123456789/collaborators \
  -H "Authorization: Bearer $FACTIFY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "invitations": [
      {
        "email": "colleague@example.com",
        "role": "Collaborator"
      },
      {
        "email": "reviewer@example.com", 
        "role": "Commenter"
      }
    ],
    "sendEmail": true
  }'
```

### Invite Existing Users by ID

```bash
curl -X POST https://api.factify.com/documents/doc_123456789/collaborators \
  -H "Authorization: Bearer $FACTIFY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "invitations": [
      {
        "userId": "user_456",
        "role": "Admin"
      }
    ],
    "sendEmail": false
  }'
```

### Response

```json
{
  "document_id": "doc_123456789",
  "successful_invitations": [
    {
      "id": "collab_111",
      "role": "Collaborator",
      "status": "invitation_pending",
      "joined_at": "2024-01-15T10:35:00Z",
      "user": {
        "id": "user_789",
        "type": "user",
        "name": "Jane Smith",
        "email": "colleague@example.com"
      }
    }
  ],
  "failed_invitations": [],
  "invited_by": {
    "id": "user_123",
    "name": "John Doe"
  }
}
```

## Step 5: Create a New Version

Let's create a new version of your document with updated content:

```bash
curl -X POST https://api.factify.com/documents/doc_123456789/versions \
  -H "Authorization: Bearer $FACTIFY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Document v2",
    "description": "Second version with additional content and improvements"
  }'
```

### Response

```json
{
  "id": "ver_555666777",
  "document_id": "doc_123456789",
  "name": "Updated Document v2",
  "description": "Second version with additional content and improvements",
  "created_at": "2024-01-15T11:00:00Z",
  "created_by": {
    "object": "user",
    "id": "user_123",
    "name": "John Doe"
  },
  "status": "done",
  "url": "https://factify.com/documents/doc_123456789/versions/ver_555666777"
}
```

## Step 6: Generate an Entry Page

Now let's create a public entry page for your document. This allows external users to access the document without needing a Factify account.

### Generate Entry Page

```bash
curl -X POST https://api.factify.com/documents/doc_123456789/entry-page \
  -H "Authorization: Bearer $FACTIFY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "template_default",
    "version_id": "ver_555666777"
  }'
```

### Response

```json
{
  "entry_page_url": "https://factify.com/entry/abc123def456"
}
```

### Access Your Entry Page

You can now share the entry page URL with anyone, and they'll be able to view your document without needing to sign up for Factify.

<Tip>
Entry pages are perfect for sharing documents with clients, stakeholders, or anyone who needs read-only access without creating an account.
</Tip>

## Step 7: Download the Entry Page Content

To programmatically access the entry page content, you can make a request to the entry page URL:

### Using cURL

```bash
curl -X GET https://factify.com/entry/abc123def456 \
  -H "Accept: application/json"
```

### Using JavaScript

```javascript
const entryPageResponse = await fetch('https://factify.com/entry/abc123def456', {
  headers: {
    'Accept': 'application/json'
  }
});

const entryPageData = await entryPageResponse.json();
console.log('Entry page content:', entryPageData);
```

### Using Python

```python
import requests

response = requests.get(
    'https://factify.com/entry/abc123def456',
    headers={'Accept': 'application/json'}
)

entry_page_data = response.json()
print('Entry page content:', entry_page_data)
```
